# Launch artifacts

Pre-written copy + plans for the Dictivo v1.0 launch. Use these as
starting points; tune them for the actual final product state.

| File | Use |
|---|---|
| `show-hn.md` | The single most valuable piece of free distribution. Title + body + canned replies. |
| `product-hunt-kit.md` | Day-after-HN PH launch: tagline, description, gallery specs, maker comment. |
| `where-your-data-lives.md` | Marketing-site page content. Drop into `034_Dictivo_Site` repo and link from privacy strip. |

## Recommended launch sequence

1. **Day 0 (silent)**: ship v1.0, verify purchase → activation → update flow with a real Lemon Squeezy production key, deploy the marketing site with real LS links.
2. **Day 1 (HN)**: post Show HN at 14:00 CET. Stay online for 6 hours. Reply to every comment.
3. **Day 2 or 3 (PH)**: launch on Product Hunt at 12:01 AM Pacific. Stay online for the first 4 hours.
4. **Week 1**: reply to anything that mentions Dictivo on Twitter / r/macapps / Hacker Newsletter.
5. **Week 2**: send one Twitter thread + one Indie Hackers post recapping the launch (numbers, what worked, what didn't).

Resist the temptation to run more channels in parallel. A solo dev cannot keep up with HN + PH + Reddit + Twitter + an email newsletter simultaneously, and partial attention to each is worse than full attention to two.
